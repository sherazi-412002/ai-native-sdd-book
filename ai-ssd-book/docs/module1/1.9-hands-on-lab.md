# Chapter 1.9: Hands-On Lab

## Introduction

The hands-on lab section provides practical, guided exercises that allow you to apply the concepts learned in the previous chapters. These labs are designed to give you real-world experience with ROS 2 development, from basic setup to complex integration scenarios.

This chapter walks you through comprehensive laboratory exercises that build upon the theoretical knowledge presented in earlier chapters, providing you with the skills needed to develop and deploy ROS 2 applications.

## Lab 1: Setting Up Your Development Environment

### Objective
Establish a complete ROS 2 development environment with all necessary tools and dependencies.

- Ubuntu 22.04 LTS or equivalent Linux distribution
- ROS 2 Jazzy/Jolt installed
- Python 3.10+ environment

## Lab 2: Creating Your First ROS 2 Package

### Objective
Create a custom ROS 2 package that implements a simple sensor node.

### Lab Steps

#### 1. Package Creation
```bash
# Create a sensor package
ros2 pkg create --build-type ament_cmake sensor_package \
    --dependencies rclcpp std_msgs sensor_msgs
```

#### 2. Implement Sensor Node
Create `src/sensor_node.cpp`:
```cpp
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>
#include <sensor_msgs/msg/laser_scan.hpp>
#include <chrono>

class SensorNode : public rclcpp::Node {
public:
    SensorNode() : Node("sensor_node"), count_(0) {
        // Create publishers
        string_publisher_ = this->create_publisher<std_msgs::msg::String>("sensor/string", 10);
        laser_publisher_ = this->create_publisher<sensor_msgs::msg::LaserScan>("sensor/laser", 10);

        // Create timer
        timer_ = this->create_wall_timer(
            std::chrono::milliseconds(1000),
            std::bind(&SensorNode::publish_sensor_data, this));
    }

private:
    void publish_sensor_data() {
        // Publish string data
        auto string_msg = std_msgs::msg::String();
        string_msg.data = "Sensor reading " + std::to_string(count_);
        string_publisher_->publish(string_msg);

        // Publish laser scan data
        auto laser_msg = sensor_msgs::msg::LaserScan();
        laser_msg.header.stamp = this->now();
        laser_msg.header.frame_id = "laser_frame";
        laser_msg.angle_min = -1.57;
        laser_msg.angle_max = 1.57;
        laser_msg.angle_increment = 0.01;
        laser_msg.range_min = 0.1;
        laser_msg.range_max = 10.0;
        laser_msg.ranges.resize(314, 1.0 + static_cast<float>(count_) * 0.1);
        laser_publisher_->publish(laser_msg);

        RCLCPP_INFO(this->get_logger(), "Published sensor data %d", count_);
        count_++;
    }

    rclcpp::Publisher<std_msgs::msg::String>::SharedPtr string_publisher_;
    rclcpp::Publisher<sensor_msgs::msg::LaserScan>::SharedPtr laser_publisher_;
    rclcpp::TimerBase::SharedPtr timer_;
    size_t count_;
};

int main(int argc, char * argv[]) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<SensorNode>());
    rclcpp::shutdown();
    return 0;
}
```

#### 3. Update CMakeLists.txt
Add to `CMakeLists.txt`:
```cmake
# Find dependencies
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(std_msgs REQUIRED)
find_package(sensor_msgs REQUIRED)

# Create executable
add_executable(sensor_node src/sensor_node.cpp)
ament_target_dependencies(sensor_node rclcpp std_msgs sensor_msgs)

# Install executable
install(TARGETS
  sensor_node
  DESTINATION lib/${PROJECT_NAME})

# Install launch directory
install(DIRECTORY
  launch
  DESTINATION share/${PROJECT_NAME})
```

#### 4. Build and Test
```bash
# Build the package
cd ~/ros2_workspace
colcon build --packages-select sensor_package

# Source workspace
source install/setup.bash

# Run sensor node
ros2 run sensor_package sensor_node
```

### Expected Outcome
You should successfully create and run a sensor node that publishes both string and laser scan data at regular intervals.

## Lab 4: Implementing Talker-Listener Pattern

### Objective

### Lab Steps

#### 2. Implement Enhanced Talker
Create `src/enhanced_talker.cpp`:
```cpp
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>
#include <chrono>
#include <random>

class EnhancedTalker : public rclcpp::Node {
public:
    EnhancedTalker() : Node("enhanced_talker"), count_(0) {
        // Create publisher
        publisher_ = this->create_publisher<std_msgs::msg::String>("chatter", 10);

        // Create timer
        timer_ = this->create_wall_timer(
            std::chrono::milliseconds(500),
            std::bind(&EnhancedTalker::publish_message, this));

        // Initialize random generator
        random_generator_.seed(std::random_device{}());
    }

private:
    void publish_message() {
        // Generate different types of messages
        std::uniform_int_distribution<int> type_dist(1, 4);
        int message_type = type_dist(random_generator_);

        auto message = std_msgs::msg::String();

        switch (message_type) {
            case 1:
                message.data = "Status update: " + std::to_string(count_);
                break;
            case 2:
                message.data = "Warning: Battery level low";
                break;
            case 3:
                message.data = "Info: System OK";
                break;
            case 4:
                message.data = "Alert: Obstacle detected";
                break;
        }

        RCLCPP_INFO(this->get_logger(), "Publishing: '%s'", message.data.c_str());
        publisher_->publish(message);
        count_++;
    }

    rclcpp::Publisher<std_msgs::msg::String>::SharedPtr publisher_;
    rclcpp::TimerBase::SharedPtr timer_;
    size_t count_;
    std::mt19937 random_generator_;
};

int main(int argc, char * argv[]) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<EnhancedTalker>());
    rclcpp::shutdown();
    return 0;
}
```

#### 3. Implement Listener with Filtering
Create `src/filtering_listener.cpp`:
```cpp
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>

class FilteringListener : public rclcpp::Node {
public:
    FilteringListener() : Node("filtering_listener") {
        // Create subscriber
        subscription_ = this->create_subscription<std_msgs::msg::String>(
            "chatter", 10,
            [this](const std_msgs::msg::String::SharedPtr msg) {
                this->process_message(msg);
            });
    }

private:
    void process_message(const std_msgs::msg::String::SharedPtr msg) {
        // Filter messages based on content
        if (msg->data.find("Warning") != std::string::npos) {
            RCLCPP_WARN(this->get_logger(), "FILTERED WARNING: %s", msg->data.c_str());
        } else if (msg->data.find("Alert") != std::string::npos) {
            RCLCPP_ERROR(this->get_logger(), "FILTERED ALERT: %s", msg->data.c_str());
        } else {
            RCLCPP_INFO(this->get_logger(), "RECEIVED: %s", msg->data.c_str());
        }
    }

    rclcpp::Subscription<std_msgs::msg::String>::SharedPtr subscription_;
};

int main(int argc, char * argv[]) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<FilteringListener>());
    rclcpp::shutdown();
    return 0;
}
```

```cpp
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>
#include <chrono>
#include <string>

class AIIntegrationNode : public rclcpp::Node {
public:
    AIIntegrationNode() : Node("ai_integration_node") {
        // Create publishers and subscribers
        ai_response_publisher_ = this->create_publisher<std_msgs::msg::String>("ai_responses", 10);
        command_subscriber_ = this->create_subscription<std_msgs::msg::String>(
            "user_commands", 10,
            [this](const std_msgs::msg::String::SharedPtr msg) {
                this->process_user_command(msg);
            });

        // Create timer for periodic AI checks
        timer_ = this->create_wall_timer(
            std::chrono::seconds(30),
            std::bind(&AIIntegrationNode::periodic_ai_check, this));
    }

private:
    void process_user_command(const std_msgs::msg::String::SharedPtr msg) {
        // In a real implementation, this would call the AI API
        RCLCPP_INFO(this->get_logger(), "Processing command: %s", msg->data.c_str());

        // Simulate AI processing
        auto response = std_msgs::msg::String();
        response.data = "AI response to: " + msg->data;

        ai_response_publisher_->publish(response);
        RCLCPP_INFO(this->get_logger(), "Published AI response");
    }

    void periodic_ai_check() {
        // Periodic AI system check
        RCLCPP_INFO(this->get_logger(), "Performing periodic AI system check");
    }

    rclcpp::Publisher<std_msgs::msg::String>::SharedPtr ai_response_publisher_;
    rclcpp::Subscription<std_msgs::msg::String>::SharedPtr command_subscriber_;
    rclcpp::TimerBase::SharedPtr timer_;
};

int main(int argc, char * argv[]) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<AIIntegrationNode>());
    rclcpp::shutdown();
    return 0;
}
```

#### 3. Create Python Script for AI Processing
Create `src/ai_processor.py`:
```python
#!/usr/bin/env python3

import rclpy
from rclpy.node import Node
from std_msgs.msg import String
import openai
import os

class AIProcessor(Node):
    def __init__(self):
        super().__init__('ai_processor')

        # Initialize OpenAI client
        api_key = os.getenv('OPENAI_API_KEY')
        if not api_key:
            self.get_logger().error("OPENAI_API_KEY environment variable not set")
            raise ValueError("OpenAI API key required")

        self.client = openai.OpenAI(api_key=api_key)

        # Create subscriptions and publishers
        self.command_subscriber = self.create_subscription(
            String, 'user_commands', self.command_callback, 10)
        self.response_publisher = self.create_publisher(String, 'ai_responses', 10)

        self.get_logger().info("AI Processor node initialized")

    def command_callback(self, msg):
        self.get_logger().info(f'Received command: {msg.data}')

        try:
            # Call OpenAI API
            response = self.client.chat.completions.create(
                model="gpt-4o",
                messages=[
                    {"role": "system", "content": "You are a helpful robot assistant."},
                    {"role": "user", "content": msg.data}
                ],
                temperature=0.7
            )

            # Extract and publish response
            ai_response = response.choices[0].message.content

            response_msg = String()
            response_msg.data = ai_response
            self.response_publisher.publish(response_msg)

            self.get_logger().info(f'AI Response: {ai_response}')

        except Exception as e:
            self.get_logger().error(f'Error processing AI request: {str(e)}')
            error_msg = String()
            error_msg.data = f"AI processing error: {str(e)}"
            self.response_publisher.publish(error_msg)

def main(args=None):
    rclpy.init(args=args)

    try:
        node = AIProcessor()
        rclpy.spin(node)
    except KeyboardInterrupt:
        node.get_logger().info('Shutting down AI processor')
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## Troubleshooting Guide

### Common Issues and Solutions

#### 1. Package Not Found
**Problem**: `ros2 run package_name node_name` fails with "package not found"
**Solution**:
```bash
# Ensure workspace is sourced
source install/setup.bash

# Verify package exists
ros2 pkg list | grep package_name
```

#### 2. Permission Denied
**Problem**: Permission denied when running nodes
**Solution**:
```bash
# Check file permissions
chmod +x src/node.cpp

# Or run with sudo (not recommended for production)
sudo ros2 run package_name node_name
```

#### 3. Network Communication Issues
**Problem**: Nodes cannot communicate
**Solution**:
```bash
# Check if nodes are running
ros2 node list

# Check topics
ros2 topic list

# Check topic information
ros2 topic info /topic_name
```
