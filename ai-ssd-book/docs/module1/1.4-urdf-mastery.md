# Chapter 1.4: URDF Mastery

## Introduction

Universal Robot Description Format (URDF) is the cornerstone of robot modeling in ROS 2. It provides a standardized XML-based format for describing robot geometry, kinematics, dynamics, and other properties. Mastering URDF is essential for anyone working with robotic systems, as it enables the creation of accurate robot models that can be used for simulation, visualization, and real-world deployment.

This chapter explores the intricacies of URDF, from basic concepts to advanced modeling techniques, with practical examples and best practices.

## What is URDF?

Universal Robot Description Format (URDF) is an XML-based format used to describe robots in ROS 2. It provides:

- **Geometric Description**: Links and joints that make up the robot structure
- **Kinematic Properties**: Joint types, limits, and transformations
- **Visual Properties**: Mesh files, colors, and materials for visualization
- **Collision Properties**: Geometry for collision detection
- **Dynamic Properties**: Mass, inertia, and other physical characteristics

## Basic URDF Structure

A basic URDF document consists of several key elements:

```xml
<?xml version="1.0"?>
<robot name="my_robot">
  <!-- Links -->
  <link name="base_link">
    <visual>
      <geometry>
        <box size="0.5 0.5 0.1"/>
      </geometry>
    </visual>
  </link>

  <!-- Joints -->
  <joint name="base_to_wheel" type="revolute">
    <parent link="base_link"/>
    <child link="wheel_link"/>
    <origin xyz="0 0 0.1"/>
    <axis xyz="0 1 0"/>
  </joint>

  <!-- Additional links and joints -->
</robot>
```

## URDF Elements Explained

### 1. Robot Element

The `<robot>` element is the root element that defines the robot:

```xml
<robot name="robot_name" xmlns:xacro="http://www.ros.org/wiki/xacro">
  <!-- Robot description goes here -->
</robot>
```

### 2. Link Elements

Links represent the physical parts of the robot:

```xml
<link name="base_link">
  <inertial>
    <mass value="1.0"/>
    <origin xyz="0 0 0"/>
    <inertia ixx="0.01" ixy="0" ixz="0" iyy="0.01" iyz="0" izz="0.01"/>
  </inertial>
  <visual>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <geometry>
      <box size="0.5 0.5 0.1"/>
    </geometry>
    <material name="blue">
      <color rgba="0 0 1 1"/>
    </material>
  </visual>
  <collision>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <geometry>
      <box size="0.5 0.5 0.1"/>
    </geometry>
  </collision>
</link>
```

### 3. Joint Elements

Joints define the relationships between links:

```xml
<joint name="joint_name" type="revolute">
  <parent link="parent_link"/>
  <child link="child_link"/>
  <origin xyz="0.1 0.2 0.3" rpy="0.1 0.2 0.3"/>
  <axis xyz="0 0 1"/>
  <limit lower="-1.57" upper="1.57" effort="100" velocity="1.0"/>
</joint>
```

## Common Joint Types

### 1. Revolute Joints
Rotational joints around a single axis:

```xml
<joint name="shoulder_joint" type="revolute">
  <parent link="base_link"/>
  <child link="shoulder_link"/>
  <origin xyz="0 0 0.1"/>
  <axis xyz="0 0 1"/>
  <limit lower="-3.14" upper="3.14" effort="100" velocity="1.0"/>
</joint>
```

### 2. Prismatic Joints
Linear joints along a single axis:

```xml
<joint name="slider_joint" type="prismatic">
  <parent link="base_link"/>
  <child link="slider_link"/>
  <origin xyz="0 0 0"/>
  <axis xyz="1 0 0"/>
  <limit lower="0" upper="0.5" effort="100" velocity="1.0"/>
</joint>
```

### 3. Fixed Joints
Non-moving connections between links:

```xml
<joint name="camera_joint" type="fixed">
  <parent link="base_link"/>
  <child link="camera_link"/>
  <origin xyz="0.1 0.0 0.2"/>
</joint>
```

## Advanced URDF Features

### 1. Xacro for Parametrized URDFs

Xacro (XML Macros) allows for reusable and parametrized URDFs:

```xml
<?xml version="1.0"?>
<robot name="my_robot" xmlns:xacro="http://www.ros.org/wiki/xacro">
  <xacro:property name="wheel_radius" value="0.1"/>
  <xacro:property name="wheel_width" value="0.05"/>

  <link name="wheel">
    <visual>
      <geometry>
        <cylinder radius="${wheel_radius}" length="${wheel_width}"/>
      </geometry>
    </visual>
  </link>
</robot>
```

### 2. Inertial Properties

Accurate inertial properties are crucial for simulation:

```xml
<link name="arm_link">
  <inertial>
    <mass value="2.0"/>
    <origin xyz="0 0 0"/>
    <inertia ixx="0.01" ixy="0" ixz="0" iyy="0.01" iyz="0" izz="0.01"/>
  </inertial>
</link>
```

### 3. Multiple Collision Shapes

Complex collision geometries can be defined:

```xml
<collision name="collision_shape">
  <origin xyz="0 0 0" rpy="0 0 0"/>
  <geometry>
    <mesh filename="package://my_robot/meshes/arm.STL"/>
  </geometry>
</collision>
```

## Complete 12-DoF Humanoid URDF Example

Here's a simplified example of a 12-DoF humanoid robot:

```xml
<?xml version="1.0"?>
<robot name="humanoid_robot" xmlns:xacro="http://www.ros.org/wiki/xacro">
  <!-- Constants -->
  <xacro:property name="body_height" value="0.5"/>
  <xacro:property name="body_width" value="0.3"/>
  <xacro:property name="arm_length" value="0.4"/>
  <xacro:property name="leg_length" value="0.5"/>

  <!-- Base link -->
  <link name="base_link">
    <visual>
      <geometry>
        <box size="${body_width} ${body_width} ${body_height}"/>
      </geometry>
      <material name="gray">
        <color rgba="0.5 0.5 0.5 1"/>
      </material>
    </visual>
    <collision>
      <geometry>
        <box size="${body_width} ${body_width} ${body_height}"/>
      </geometry>
    </collision>
  </link>

  <!-- Head link -->
  <link name="head_link">
    <visual>
      <geometry>
        <sphere radius="0.1"/>
      </geometry>
      <material name="white">
        <color rgba="1 1 1 1"/>
      </material>
    </visual>
  </link>

  <!-- Head joint -->
  <joint name="head_joint" type="revolute">
    <parent link="base_link"/>
    <child link="head_link"/>
    <origin xyz="0 0 ${body_height/2 + 0.1}"/>
    <axis xyz="0 0 1"/>
    <limit lower="-0.5" upper="0.5" effort="100" velocity="1.0"/>
  </joint>

  <!-- Left arm joints -->
  <joint name="left_shoulder_joint" type="revolute">
    <parent link="base_link"/>
    <child link="left_upper_arm"/>
    <origin xyz="-${body_width/2} 0 ${body_height/2}"/>
    <axis xyz="0 0 1"/>
    <limit lower="-1.57" upper="1.57" effort="100" velocity="1.0"/>
  </joint>

  <!-- Right arm joints -->
  <joint name="right_shoulder_joint" type="revolute">
    <parent link="base_link"/>
    <child link="right_upper_arm"/>
    <origin xyz="${body_width/2} 0 ${body_height/2}"/>
    <axis xyz="0 0 1"/>
    <limit lower="-1.57" upper="1.57" effort="100" velocity="1.0"/>
  </joint>

  <!-- Additional joints for arms, legs, etc. would follow -->

</robot>
```

## URDF Visualization and Tools

### 1. RViz Integration

URDF models can be visualized in RViz:

```bash
# Launch RViz with URDF
ros2 launch urdf_tutorial display.launch.py model:=/path/to/robot.urdf
```

### 2. URDF Validation

Validate URDF files using the `check_urdf` tool:

```bash
# Check URDF validity
check_urdf /path/to/robot.urdf

# Visualize URDF in Rviz
rviz2 -d /path/to/robot.urdf
```

### 3. URDF Editor Tools

Several tools are available for creating and editing URDF files:
- **ROS 2 URDF Editor**: GUI-based editors
- **Blender**: 3D modeling with URDF export plugins
- **SolidWorks**: CAD software with URDF export capabilities
- **FreeCAD**: Open-source CAD with URDF export

## Best Practices for URDF Modeling

### 1. Proper Link Naming

Use descriptive and consistent naming conventions:

```xml
<!-- Good -->
<link name="left_arm_upper_link"/>
<link name="right_leg_lower_link"/>

<!-- Avoid -->
<link name="link_1"/>
<link name="part_a"/>
```

### 2. Accurate Inertial Properties

Calculate and specify realistic inertial properties:

```xml
<link name="arm_segment">
  <inertial>
    <mass value="1.5"/>
    <origin xyz="0 0 0.1"/>
    <inertia ixx="0.02" ixy="0" ixz="0" iyy="0.02" iyz="0" izz="0.01"/>
  </inertial>
</link>
```

### 3. Collision Geometry Simplification

Use simplified collision shapes for performance:

```xml
<visual>
  <geometry>
    <mesh filename="package://robot/meshes/arm.obj"/>
  </geometry>
</visual>
<collision>
  <geometry>
    <cylinder radius="0.05" length="0.4"/>
  </geometry>
</collision>
```

### 4. Parameterized Models

Use Xacro macros for reusability:

```xml
<xacro:macro name="wheel" params="prefix">
  <link name="${prefix}_wheel">
    <visual>
      <geometry>
        <cylinder radius="0.1" length="0.05"/>
      </geometry>
    </visual>
  </link>
</xacro:macro>
```
